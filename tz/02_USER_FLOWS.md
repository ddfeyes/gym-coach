# User Flows — Сценарії взаємодії

## UF-001: Перший запуск (онбординг)

### Передумови
- Юзер має Telegram
- Юзер знайшов бота і натиснув /start

### Потік

```
1. Юзер натискає /start в Telegram
2. Бот відправляє привітальне повідомлення:
   "Привіт! Я Body Coach AI — твій персональний тренер, дієтолог 
   і health-коуч. Давай познайомимось і я створю для тебе 
   ідеальну програму 💪"
3. Бот відправляє кнопку "Відкрити додаток" (Mini App)
4. Відкривається Mini App → екран онбордингу
5. Крок 1/6 — Базова інфо:
   - Ім'я (текстове поле)
   - Стать (вибір: Жінка / Чоловік)
   - Вік (число)
   - Зріст (число, см)
   - Поточна вага (число, кг)
6. Крок 2/6 — Тренувальний досвід:
   - Рівень (вибір: Початківець <6 міс / Середній 6міс-2р / Досвідчений 2р+)
   - Скільки днів на тиждень готова тренуватись (вибір: 2/3/4/5/6)
   - Скільки часу на тренування (вибір: 45хв / 60хв / 75хв / 90хв)
7. Крок 3/6 — Обладнання:
   - Де тренуєшся (вибір: Повноцінний зал / Домашній зал / Тільки гантелі / Без обладнання)
   - Список доступного обладнання (чекліст: штанга, гантелі, тренажери, кабельна рама, турнік, брусся, гумки, TRX)
8. Крок 4/6 — Здоров'я:
   - Чи є травми або хронічні проблеми? (так/ні)
   - Якщо так → текстове поле: опиши (наприклад: "грижа L5-S1", "біль в правому плечі")
   - AI парсить текст і створює structured записи в injuries
   - Чи є обмеження по вправах? (текстове поле, опціонально)
9. Крок 5/6 — Цілі:
   - Головна ціль (вибір: Набір м'язів / Схуднення / Сила / Здоров'я / Рекомпозиція)
   - Додаткові цілі (мульти-вибір: Покращити поставу / Більше енергії / Зменшити біль / Покращити гнучкість / Підготовка до змагань)
10. Крок 6/6 — Жіноче здоров'я (якщо стать = Жінка):
    - Чи хочеш трекати менструальний цикл? (так/ні)
    - Якщо так:
      - Середня довжина циклу (число, дефолт 28)
      - Дата початку останніх місячних (date picker)
    - Якщо ні → пропускаємо, але зберігаємо що відмовилась (може ввімкнути пізніше)
11. Екран підтвердження:
    - Показує summary всіх даних
    - Кнопка "Все вірно, створити програму"
    - Кнопка "Повернутись і виправити"
12. AI генерує тренувальну програму (loading screen: "Створюю твою програму...")
13. Редірект на головний екран з готовою програмою
```

### Валідація
- Вік: 14-80
- Зріст: 100-250 см
- Вага: 30-300 кг
- Дні тренувань: 2-6
- Обов'язкові поля: ім'я, стать, вік, зріст, вага, рівень, дні, обладнання, ціль

### Edge cases
- Юзер закрив Mini App посеред онбордингу → зберігати прогрес, продовжити з того ж кроку
- Юзер ввів травму нечітко → AI робить follow-up питання в чаті після онбордингу
- Юзер вибрав "без обладнання" → програма тільки з bodyweight вправами

## UF-002: Перегляд тренувальної програми

### Передумови
- Онбординг завершено
- Програма згенерована

### Потік

```
1. Юзер відкриває Mini App → таб "Training"
2. Бачить активну програму:
   - Назва ("Hypertrophy Block 1 — Week 3")
   - Тиждень X з Y
   - Список тренувальних днів:
     ┌─────────────────────────────────┐
     │ 📅 День 1 — Push (Груди, Плечі, Тріцепс)  │
     │    6 вправ · ~60 хв                         │
     │    [Почати тренування]                       │
     ├─────────────────────────────────┤
     │ 📅 День 2 — Pull (Спина, Біцепс)           │
     │    6 вправ · ~55 хв                         │
     │    ✅ Виконано 25.03                         │
     ├─────────────────────────────────┤
     │ 📅 День 3 — Legs (Ноги, Сідниці)           │
     │    7 вправ · ~70 хв                         │
     │    [Почати тренування]                       │
     └─────────────────────────────────┘
3. Натиснувши на день → розкривається список вправ:
   ┌─────────────────────────────────┐
   │ 1. Жим штанги лежачи             │
   │    4×8-10 · Рекомендація: 40кг    │
   │    Минулого разу: 37.5кг × 10,10,9,8  │
   │                                   │
   │ 2. Жим гантелей на похилій        │
   │    3×10-12 · Рекомендація: 14кг   │
   │    Минулого разу: 12кг × 12,11,10 │
   │    ...                            │
   └─────────────────────────────────┘
4. На кожній вправі:
   - Кнопка "ℹ️" → опис вправи, техніка, помилки
   - Кнопка "🔄" → замінити вправу (AI пропонує альтернативи)
```

## UF-003: Виконання тренування

### Передумови
- Є активна програма
- Юзер натиснув "Почати тренування"

### Потік

```
1. Юзер натискає "Почати тренування" на конкретному дні
2. Система перевіряє daily checkin:
   - Якщо чекін сьогодні не зроблений → показує quick checkin form:
     - Як спала? (1-10, slider)
     - Енергія зараз (1-10, slider)
     - Крепатура (1-10, slider)
     - День циклу (auto-calculated, кнопка "скоригувати")
   - Якщо чекін є → пропускаємо
3. AI аналізує чекін:
   - Якщо все ОК → "Гарний день для тренування! Поїхали 💪"
   - Якщо проблеми → пропонує адаптацію:
     "Бачу що енергія 3/10 і ти в лютеальній фазі. 
     Пропоную зменшити ваги на 15% і прибрати останню 
     ізоляцію. Як тобі?"
     [Прийняти зміни] [Тренуватись як заплановано]
4. Починається тренування:
   - Таймер загального часу тренування
   - Показується перша вправа:
     ┌─────────────────────────────────┐
     │ Жим штанги лежачи                │
     │ Рекомендація: 40кг × 8-10        │
     │ Минулого разу: 37.5кг            │
     │                                   │
     │ Підхід 1:                         │
     │ Вага: [  40  ] кг                 │
     │ Повтори: [  10  ]                 │
     │ RPE: [😊 легко] [😐 норм] [😤 важко] [🔥 на межі] │
     │                                   │
     │ [✅ Записати підхід]               │
     │                                   │
     │ ⏱️ Відпочинок: 2:00 (рекомендовано)│
     └─────────────────────────────────┘
5. Після кожного підходу:
   - Підхід зберігається
   - Запускається таймер відпочинку (рекомендований час залежить від типу вправи)
   - Показується наступний підхід
   - Якщо RPE = "на межі" на 2+ підходах → AI попередження:
     "Занадто важко? Можеш зменшити вагу на 5-10%"
6. Після всіх підходів вправи → перехід до наступної
7. Кнопка "⏭️ Пропустити вправу" + причина (обладнання зайнято / біль / немає сил)
   - Якщо "біль" → відкривається pain journal entry
   - Якщо "обладнання зайнято" → AI пропонує замінник
8. Після останньої вправи → екран завершення:
   ┌─────────────────────────────────┐
   │ 🎉 Тренування завершено!         │
   │                                   │
   │ Час: 58 хвилин                    │
   │ Вправ: 6/6                        │
   │ Загальний об'єм: 12,450 кг        │
   │                                   │
   │ Як загалом себе почуваєш?         │
   │ [😊] [😐] [😤] [💀]              │
   │                                   │
   │ Нотатки (опціонально):            │
   │ [                              ]  │
   │                                   │
   │ [Зберегти]                        │
   └─────────────────────────────────┘
9. AI дає фідбек:
   "Відмінне тренування! Ти збільшила вагу на жимі з 37.5 до 40кг — 
   це прогрес! На наступному тренуванні спробуємо утримати 40кг 
   на всі 4 підходи. Присідання йдуть стабільно, 
   ще тиждень і підвищимо."
```

### Edge cases
- Юзер закрила додаток посеред тренування → зберігати стан, продовжити при відкритті
- Юзер хоче додати вправу якої нема в програмі → "Додати вправу" → пошук по бібліотеці
- Юзер пропустила тренувальний день → при наступному відкритті AI питає: "Ти пропустила Pull день у вівторок. Хочеш виконати його сьогодні або пропустити і продовжити за розкладом?"
- Юзер хоче змінити порядок вправ → drag and drop або кнопки ↑↓

## UF-004: Щоденний чекін

### Потік

```
1. Кожен день о 8:00 (настроюваний час) → push notification від бота:
   "Доброго ранку! Як ти сьогодні? Заповни швидкий чекін 🌅"
   [Заповнити]
2. Відкривається форма (можна в боті або в Mini App):
   - Як спала? (1-10)
   - Скільки годин сну? (число)
   - Енергія (1-10)
   - Стрес (1-10)
   - Крепатура (1-10)
   - Настрій (1-10)
   - Вага зранку (опціонально, кг)
   - День циклу (auto, можна скоригувати)
   - Біль/дискомфорт? (так/ні → якщо так, pain journal)
   - Нотатки (текст, опціонально)
3. Зберігається в daily_checkins
4. AI може відреагувати:
   - "Бачу що стрес 9/10 третій день поспіль. Може сьогодні замість 
     важкого тренування зробимо легку йогу або прогулянку?"
   - Або мовчить якщо все ок (не спамити кожен день)
```

### Правила AI-реакції на чекін
- Реагує тільки якщо є проблема (енергія <4, стрес >7, сон <5 годин, крепатура >8)
- Реагує на патерни (3+ дні поганого сну поспіль)
- Не реагує якщо все нормально (щоб не дратувати)

## UF-005: Запис болю (Pain Journal)

### Потік

```
1. Юзер може записати біль:
   a) Під час тренування (кнопка "Пропустити → Біль")
   b) В daily checkin ("Є біль? → Так")
   c) В окремому розділі "Pain Journal" в Mini App
2. Форма запису:
   - Де болить? (тіло-map або вибір: плече, коліно, поперек, тощо)
   - Яка сторона? (ліва / права / обидві / центр)
   - Тип болю (вибір: гострий / тягнучий / ниючий / пекучий / поколювання)
   - Інтенсивність (1-10)
   - Коли виникає (мульти-вибір: під час вправи / після тренування / вранці / постійно / при русі)
   - При якій вправі? (вибір з поточної програми, опціонально)
   - Нотатки
3. Зберігається
4. AI аналізує:
   - Перший запис → "Зрозуміла, буду слідкувати за цим"
   - Повторний запис тієї ж зони → "Це вже третій раз коли болить 
     праве плече після жиму. Пропоную: 1) Замінити жим штанги 
     на жим гантелями нейтральним хватом 2) Додати розминку 
     ротаторної манжети 3) Якщо не пройде за тиждень — зверніся 
     до ортопеда"
```

## UF-006: Зміна програми

### Тригери

```
1. Автоматичний (AI ініціює):
   - Закінчився запланований блок (4-6 тижнів)
   - Deload тиждень (кожні 4-6 тижнів)
   - Стагнація (вага не росте 3+ тижні)
   - Систематичний біль
   
2. Юзер ініціює:
   - "Хочу змінити програму"
   - "Мені нудно"
   - "Я тепер ходжу 4 дні замість 3"
   - "Я в подорожі, нема зали"
```

### Потік (автоматичний)

```
1. AI аналізує дані за блок
2. Відправляє повідомлення:
   "Блок 1 завершено! Ось що я бачу за 5 тижнів:
   ✅ Жим: 35кг → 42.5кг (+21%)
   ✅ Присід: 50кг → 60кг (+20%)
   ⚠️ Тяга: 60кг → 62.5кг (+4%) — повільний прогрес
   
   Пропоную Блок 2 з фокусом на спину і задню поверхню. 
   Також хочу додати більше односторонніх вправ.
   [Показати нову програму] [Обговорити зміни]"
3. Генерує нову програму
4. Юзер може попросити корекції
5. Підтвердження → стара програма archived, нова active
```

## UF-007: Чат з агентом (вільне спілкування)

### Потік

```
1. Юзер відкриває чат (через бота або таб "Chat" в Mini App)
2. Пише будь-що:
   - "Як правильно робити румунську тягу?"
   - "Чому мені болять коліна при присіді?"
   - "Я сьогодні дуже втомлена, чи варто тренуватись?"
   - "Що таке RPE?"
   - "Заміни мені жим на щось без штанги"
   - "Я хочу тренуватись 5 днів замість 4"
3. Router визначає тему:
   - training_technique → підключає training module prompt + exercise library
   - training_program → підключає training module prompt + current program + logs
   - pain_injury → підключає training module + pain journal
   - motivation → підключає psychology module
   - nutrition → підключає nutrition module (або "цей модуль ще в розробці")
   - general → базовий prompt
4. Формує контекст:
   - Base prompt + relevant module prompt
   - User profile
   - Relevant data (останні тренування, біль-журнал, тощо)
5. Claude відповідає
6. Conversation зберігається в ai_conversations
```

### Обмеження
- Агент НЕ ставить медичних діагнозів
- При серйозних симптомах → "Зверніся до лікаря"
- Не рекомендує стероїди або заборонені речовини

## UF-008: Трекінг вимірів тіла

### Потік

```
1. Раз на 2 тижні → нотифікація: "Час виміряти прогрес! 📏"
2. Юзер відкриває "Measurements" в Mini App
3. Вводить:
   - Вага
   - Обхват грудей, талії, стегон, рук, ніг
   - Може додати фото (фронт, бік, спина)
4. Система показує порівняння з минулим разом:
   - Дельта по кожному виміру
   - Графік тренду ваги
   - Порівняння фото side-by-side (якщо є)
5. AI коментує: "За місяць: вага +1.2кг, талія без змін, 
   руки +0.5см. Чудовий показник — ти набираєш м'язову масу 
   без зайвого жиру 🎯"
```

## UF-009: Замінник вправи (на ходу)

### Потік

```
1. Під час тренування юзер натискає "🔄" на вправі
2. Вибирає причину:
   - Обладнання зайнято
   - Біль/дискомфорт
   - Не подобається вправа
   - Інше
3. AI пропонує 2-3 альтернативи:
   "Замість жиму штанги лежачи можу запропонувати:
   1. Жим гантелей лежачи — найближчий аналог
   2. Жим в Смітті — якщо хочеш стабільність
   3. Жим в тренажері Hammer — якщо болить плече"
4. Юзер вибирає → заміна в поточному тренуванні
5. Запитання: "Замінити тільки сьогодні чи назавжди?"
```
